PACMAN: ASCII Edition

Close to my heart, one of my first big coding assignments was re-creating the classic game of Pacman. My game was equipped with hidden levels including a "Space Invader" level where Pacman had to battle a fleet of ghosts. Using 3 months of C++ experience, some incredible ASCII art and a rudimentary AI algorithm I recreated the childhood classic. This project solidified my knowledge of classes and helped me gain an understanding of data abstraction. It also taught me the importance of writing clean modular code.

Tool of choice: C++

Connectivity Mapping

Connectivity Mapping is a method for revealing the link between DNA-encoded information and the expressed phenotype using genome-wide gene expression. It is based on the Connectivity Map. I am working on developing a statistical model to find the relationship between drugs and their effects on gene expression. I am currently investigating several machine learning algorithms such as variational autoencoders to attempt to characterise the magnitude of microarray data available. If you are interested in finding out more information about the project or reading an abstract from the paper I am working on, contact me!

Tool of choice: R

Codename: Bomb33

Well, Binary Bomb33 if we are being specific.

The mission: Analyse and solve programs written in assembly language without exploding the bomb.
The bomb defusal manual?: An architecture manual for AMD64.

This project required a solid understanding of how C language expressions are translated into operations on memory, constants and registars. This involved understanding how complilers re-write pre processed source code into assembler source code. I also learned an arsenal of debugging skills including how to use the glorious GNU Debugger (gdb) which helped me get through the bomb levels unscathed.

Tool of choice: Assembly Language

Polymorphic 2D Arrays

This was a two-part project created to get the class thinking more about programming technique. Using Hanson's C Interfaces and Implementations we implemented two dimensional, polymorphic, unboxed arrays. Using this implementation, we then created programs to check for valid sudoku solutions (where solutions were represented by a portable graymap). Additionally, we used a modified version of the 2D array to write a program capable of removing black edges from an image.

Tool of choice: C

Riding the (MBTA) Rails

Using the JSON API and the Google Maps API, I implemented an application that showed all the stops on the MBTA (Massachusetts Bay Transportation Autority) red line. Additionally, it provided information on arrival times, distances to each red line stop using the Haversine formula and attractions near the user based on the user's geolocation. I also wrote a webserver using Heroku, Node.js and the Express web framework to host the MBTA data on train times.

Tool of choice: HTML, CSS and JavaScript

Arith: Rudimentary Image Compression

This project was designed yo give us experience working with low level machine representation of numbers. We wrote a program taht takes a PPM image and compresses the image by transforming color spaces and discarding information not easily seen by the human eye (lossy compression). We also gained a bit of an understanding of how broadcast color TV works by creating a bijection between RGB and component video color spaces. This project required a good understanding of bitwise operations, two's complement and the horrors of floating-point arithmetic.

Tool of choice: C

The Universal Machine

We wrote a software implementation of simple virtual machine in orde rto understand virtual-machine code (and by extension machine code). The UM had eight general purpose registers, an inout/output device and a 32-bit program counter. After implemetation, we improved the performance of the UM emulator by using the profiling tool kcachegrind. And lastly, we implemented an RPN calculator using UM assembly code.

Tool of choice: C, kcachegrind

Security Vulnerability

The class was tasked with identifying 3 vulnerabilies in a given website. We performed the analysis first using "black box" testing (without the source code), and then we performed an "white box" testing (analysis of the code). Issues found included cross site scripting, data overflow and noSQL injections. This is a copy of the report I wrote for that project.

Tool of choice: curl and Burp Suite
